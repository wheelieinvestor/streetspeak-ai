# v0.1 Release Notes Draft

## What works now

- Local mock trading desk with onboarding, mocked portfolio/quotes, exact confirmation gates, local redacted audit timeline, and receipt export.
- Robinhood read-only fixture explorer using committed static fixtures only.
- Real Robinhood MCP read-only boundary and panel that can use an externally managed runtime client.
- Redacted Robinhood MCP smoke harness for safe read-only connectivity checks.

## Mock-only

- Mock portfolio, mock quotes, mock order tickets, mock order review, and mock submission receipts.
- Generic confirmations remain rejected.
- Exact mock confirmation phrase/code remains required before mock submission.

## Fixture-only

- Robinhood fixture explorer.
- Fixture account summary, buying power, portfolio, positions, quotes, orders, tradability, and search.
- Fixture values are static demo data, not real broker data.

## Read-only MCP boundary

- Allowed read-only MCP checks: `get_accounts`, `get_portfolio`, `get_equity_positions`, `get_equity_quotes`, `get_equity_orders`, `get_equity_tradability`, and `search`.
- Smoke harness prints only redacted summaries.
- MCP auth/config stays externally managed outside the repo/app.
- Real read-only data is not persisted to local storage by default.

## Not built yet

- Robinhood order review.
- Robinhood order placement.
- Robinhood cancel order.
- Live broker execution.
- Broker login UI.
- Deployment, production, payment, Discord/X, Public, or ElevenLabs integrations.
- Investment-advice, trade-recommendation, signal, or autonomous trading behavior.

## Safety guarantees

- No live trading.
- No order review, order placement, or cancel order for Robinhood.
- No real order ticket submission.
- No broker secrets, API keys, tokens, MCP URLs, or raw auth config committed.
- No real account IDs, portfolio values, holdings, order IDs, raw MCP output, or screenshots with real account data committed.
- Live trading flags remain false/unavailable.
