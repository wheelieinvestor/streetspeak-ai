# v0.1 Demo Checklist

Use this checklist for a local reviewer demo. StreetSpeak AI v0.1 is mock-first, fixture-only for committed demo data, and read-only/unavailable by default for the externally managed Robinhood MCP panel. It is not investment advice and does not place live broker orders.

## Setup

- Install dependencies: `pnpm install`.
- Run the app: `pnpm --filter @streetspeak-ai/web dev`.
- Open the local Vite URL.

## Demo Flow

- Accept onboarding.
  - Confirm the acknowledgements say StreetSpeak AI is not investment advice.
  - Confirm the acknowledgements say the current version is mock-only.
  - Confirm no live broker order will be placed.
- Run a portfolio command: `show my portfolio`.
  - Confirm the answer and portfolio panel use static mock portfolio data.
- Run a quote command: `what is HOOD trading at`.
  - Confirm the quote is static mock market data, not real market data.
- Create a mock ticket: `buy 5 HOOD`.
  - Confirm the app shows parsed intent, mock order ticket, safety review, and an exact confirmation challenge.
- Reject a generic confirmation.
  - Enter `yes`.
  - Confirm no mock submission appears and no broker order is placed.
- Complete the exact-code mock confirmation.
  - Enter the full challenge phrase and code shown by the app.
  - Confirm the result says mock submission only and no live broker order was placed.
- Export a receipt.
  - Use `Copy receipt Markdown` or `Download receipt JSON`.
  - Confirm the export is local-only and includes `No live broker order was placed.`
- View the Robinhood fixture explorer.
  - Confirm the section title is `Robinhood Read-Only Fixture Explorer`.
  - Confirm account summary, buying power, portfolio, positions, quote lookup, order history, tradability, and symbol search are labeled fixture-only/static.
- View the real Robinhood MCP read-only panel.
  - Confirm it is separate from the mock trading desk and fixture explorer.
  - Confirm connection status is visible.
  - Confirm read-only mode is visible.
  - Confirm default state is unavailable/unconfigured unless an external runtime client is injected.
  - Confirm real Robinhood verified read-only is not claimed unless a redacted smoke test was actually run.
  - Confirm no live trading, no order review, no order placement, and no cancel order are visible.
  - Confirm there are no buy, sell, review, place, execute, or cancel buttons.
  - Confirm unavailable/unconfigured state is safe when no external MCP client is injected.
- Run the safe smoke harness if needed: `pnpm smoke:robinhood:readonly`.
  - Copy only redacted summary lines.
  - Never copy raw account IDs, portfolio values, holdings, order IDs, quote prices, raw MCP output, secrets, tokens, authorization headers, or MCP URLs.
- Prepare safe screenshots.
  - Use mock or fixture data, or the real read-only panel with real account data absent/redacted.
  - Do not screenshot real account data or raw MCP output.
- Confirm no live broker connection exists.
  - No broker login button.
  - No API key, token, credential, or MCP URL field.
  - Adapter transport is `none`.
  - Credentials required is `false`.
  - Order review, order placement, cancel order, and live execution are unavailable.

## Reset

- Use `Reset demo state` for transient fields.
- Use `Clear audit timeline` for persisted redacted audit events.
- Use `Reset all local demo data` before screenshots or another clean demo pass.
