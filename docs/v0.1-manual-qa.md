# v0.1 Manual QA

Use this before public demo screenshots or a v0.1 release note. Keep QA local. Do not deploy, connect production services, commit secrets, or run live broker integrations.

## Required Commands

Run from the repo root:

```sh
pnpm install
pnpm format
pnpm build
pnpm test
pnpm lint
pnpm typecheck
pnpm smoke:robinhood:readonly
git diff --check
```

## Browser QA

Start the app:

```sh
pnpm --filter @streetspeak-ai/web dev
```

Verify:

- First-run onboarding blocks the demo until every acknowledgement is checked.
- Top badges show `Mock Only` and `No Live Trading`.
- Local settings show mock mode locked on and live trading unavailable/disabled.
- Browser voice status is clear. If unsupported, typed commands still work. If supported, transcripts route through the same mock parser as typed input.
- `show my portfolio` returns static mock portfolio data.
- `what is HOOD trading at` returns a static mock quote.
- `buy 5 HOOD` creates a mock order ticket, safety review, and exact confirmation challenge.
- `yes` is rejected as a generic confirmation and creates no mock broker response.
- The exact displayed phrase/code records a mock submission only.
- Receipt Markdown/JSON export is local-only and states `No live broker order was placed.`
- Audit JSON export is local-only, redacted, and clearable.
- `buy $500 of HOOD` is unsupported, performs no share conversion, and creates no final ticket.
- `Reset demo state`, `Reset onboarding`, `Clear audit timeline`, and `Reset all local demo data` work.

## Robinhood Panel QA

Fixture explorer:

- Section title is `Robinhood Read-Only Fixture Explorer`.
- Values are labeled fixture/static and not real account, broker, or market data.
- Adapter transport is `none`.
- Credentials required is `false`.
- Live execution, order review, order placement, and cancel order are `false`.
- No broker login, credential field, token field, MCP URL field, buy, sell, review, place, execute, or cancel action is rendered.

Real MCP read-only panel:

- Section title is `Real Robinhood MCP Read-Only Connection`.
- Panel is visibly separate from the mock trading desk and fixture explorer.
- Default connection state is unavailable/unconfigured without an injected external runtime client.
- Transport is `externally_managed_mcp`.
- Credentials are externally managed and not stored by StreetSpeak.
- Credential fields required is `0`.
- Storage policy is `in_memory_only`.
- Live execution, order review, order placement, and cancel order are `false`.
- Refresh controls are read-only queries only.
- No buy, sell, review, place, execute, or cancel action is rendered.

## Screenshot Checklist

Safe screenshot candidates:

- Onboarding acknowledgement.
- Mock desk with `show my portfolio`.
- Mock desk with `what is HOOD trading at`.
- Exact confirmation challenge for `buy 5 HOOD`.
- Generic confirmation rejection after `yes`.
- Mock receipt export controls.
- v0.1 status panel and safety checklist.
- Fixture explorer with static data labels.
- MCP panel in unavailable/unconfigured state.

Do not screenshot:

- Real account IDs or account numbers.
- Real portfolio values, buying power, cash, holdings, positions, order IDs, or raw order history.
- Raw live quote prices from an MCP-backed account.
- Raw MCP output.
- Credentials, tokens, authorization headers, API keys, MCP URLs, auth config, or local client module paths.
- Browser tabs or terminal windows exposing secrets or real account data.

## Smoke Harness Redaction Checks

- Default smoke run must be unavailable/unconfigured.
- `raw payload included` must be `false`.
- Live execution, order review, order placement, and cancel order must print `false`.
- If an external client is used later, publish only allowed redacted summary lines from [robinhood-readonly-smoke-test.md](robinhood-readonly-smoke-test.md).
- Never copy raw account data, holdings, portfolio values, order IDs, quote prices, raw MCP output, credentials, tokens, API keys, or MCP URLs into docs, screenshots, PR bodies, or handoffs.

## Safety Boundary Checks

- No live trading.
- No order review implementation.
- No order placement implementation.
- No cancel-order implementation.
- No real order submission.
- No broker login UI.
- No secrets, API keys, tokens, MCP URLs, auth config, or credentials in tracked files.
- No real account data, portfolio values, holdings, order IDs, screenshots, or raw MCP output in tracked files.
- Generic confirmations remain rejected.
- Exact mock confirmation phrase/code remains required.
- Mock mode still works.
- Live trading flags remain false/unavailable.
- No investment advice or trade recommendation behavior is added.
