# v0.1 Public Demo Guide

This is a 2-3 minute public walkthrough for StreetSpeak AI v0.1. It demonstrates the local mock trading desk, exact confirmation safety gate, local receipt/audit exports, fixture explorer, and read-only MCP boundary. It does not connect to a live broker or place trades.

## Setup

```sh
pnpm install
pnpm --filter @streetspeak-ai/web dev
```

Open the local Vite URL shown in the terminal.

Optional safe smoke harness check:

```sh
pnpm smoke:robinhood:readonly
```

Without an externally managed MCP client, the expected smoke result is unavailable/unconfigured. Do not add credentials, tokens, MCP URLs, or a live broker client for a public demo.

## Walkthrough

1. Accept onboarding.
   Confirm the modal says StreetSpeak AI is not investment advice, is mock-only today, is not affiliated with Robinhood, Public, ElevenLabs, or any broker, and will not place a live broker order.

2. Show the mock desk status.
   Confirm the top badges say `Mock Only` and `No Live Trading`, and the v0.1 status panel says live trading is unavailable.

3. Run `show my portfolio`.
   Explain that the portfolio is static mock data from the repo, not a broker account.

4. Run `what is HOOD trading at`.
   Explain that the quote is static mock data, not real market data.

5. Run `buy 5 HOOD`.
   Show parsed intent, mock order ticket, safety review, and the exact confirmation phrase/code.

6. Enter `yes`.
   Confirm the app rejects the generic confirmation and does not create a mock broker response.

7. Enter the exact phrase/code shown by the app.
   Confirm the result is mock submission only and says no live broker order was placed.

8. Export a receipt.
   Use `Copy receipt Markdown` or `Download receipt JSON`. Confirm the receipt is local-only and states `No live broker order was placed.`

9. Show the audit timeline.
   Confirm redacted events are stored in local browser storage only and can be downloaded or cleared.

10. Show the `Real Robinhood MCP Read-Only Connection` panel.
    Confirm it is separate from the mock trading desk, unavailable/unconfigured by default, read-only, and has no broker login, credential field, MCP URL field, order review, order placement, cancel order, or live execution.

11. Show the `Robinhood Read-Only Fixture Explorer`.
    Confirm every account, portfolio, position, quote, order-history, tradability, and search value is labeled fixture/static and not real account, broker, or market data.

12. Run `buy $500 of HOOD`.
    Confirm notional orders are unsupported in v0.1, no share conversion is performed, and no final ticket is created.

## What To Say

- StreetSpeak AI v0.1 is a local mock demo for self-directed workflows.
- Browser voice works where browser-native speech recognition is supported; typed input is the reliable fallback.
- Generic confirmations are rejected.
- Exact phrase/code confirmation is required even for mock submission.
- Receipt and audit exports are local-only.
- The fixture explorer uses static fake data.
- The MCP panel is read-only and unavailable by default unless an external runtime client is injected.
- Real Robinhood read-only verification should not be claimed unless a redacted smoke test was actually run.

## What Not To Say

- Do not say the app places trades.
- Do not say Robinhood is connected by default.
- Do not say order review is implemented.
- Do not say live execution is implemented.
- Do not say the app recommends trades or gives investment advice.
- Do not imply affiliation with Robinhood, Public, ElevenLabs, or any broker or voice provider.

## What Not To Screenshot

- Real account IDs or account numbers.
- Real portfolio values, buying power, cash, holdings, positions, or order IDs.
- Raw MCP output.
- Credentials, tokens, authorization headers, API keys, MCP URLs, or auth config.
- Any browser tab, terminal, or file that exposes secrets or real account data.
