# v0.1 Release Checklist

StreetSpeak AI v0.1 should ship as a local, mock-first trading desk with a fixture-only Robinhood readiness panel and a strictly read-only Robinhood MCP panel that is unavailable by default unless externally configured at runtime. Do not claim order review, order placement, cancel order, live broker execution, investment advice, or trade recommendation behavior.

## Local Setup

- Run `pnpm install`.
- Run `pnpm format`.
- Run `pnpm build`.
- Run `pnpm test`.
- Run `pnpm lint`.
- Run `pnpm typecheck`.
- Run `pnpm smoke:robinhood:readonly`.
- Run `git diff --check`.
- Safe default smoke harness expectation: unavailable/unconfigured.
  - Expected without external MCP client: unavailable/unconfigured.
  - If external MCP is used, record only redacted summary lines from `docs/robinhood-readonly-smoke-test.md`.

## Safety Checks

- First-run onboarding must be required before the mock demo opens.
- Mock mode must be locked on.
- Live trading must be unavailable.
- Generic confirmations such as `yes`, `do it`, or `confirmed` must be rejected.
- The exact confirmation phrase and code must be required before mock submission.
- Unsupported notional commands must not create final tickets or convert dollars into shares.
- Receipts and audit exports must remain local-only.
- Robinhood fixture adapter status must show `transport: none`.
- Robinhood MCP adapter status must show `transport: externally_managed_mcp` and unavailable/unconfigured when no runtime client exists.
- Robinhood adapter status must show `requiresCredentials: false`.
- Robinhood order review, placement, cancel order, and live execution must remain unavailable.
- Real Robinhood read-only data must not be persisted to local storage by default.
- Real Robinhood verified read-only must not be claimed unless a redacted smoke test was actually performed.
- No broker login, credential field, token field, API key field, or MCP URL field should render.
- No investment advice or trade recommendation behavior should be present.

## What Not To Screenshot

- Real account IDs or account numbers.
- Real portfolio values, buying power, cash, holdings, positions, order IDs, or raw order history.
- Raw live quote prices from MCP-backed data.
- Raw MCP output.
- Credentials, tokens, authorization headers, API keys, MCP URLs, auth config, or local client module paths.

## Public Claim Boundaries

- Do not claim live trading.
- Do not claim a working real Robinhood MCP connection unless a redacted read-only smoke test was actually performed.
- Do not publish real account data, holdings, order IDs, portfolio values, screenshots, raw MCP output, or real market data snapshots.
- Do not claim broker login support.
- Do not claim order review, order placement, or cancel-order support for Robinhood.
- Do not claim investment advice, trade recommendations, signals, or automated trading.
- Do describe the project as local-first, mock-first, self-directed, auditable, and safety-gated.
