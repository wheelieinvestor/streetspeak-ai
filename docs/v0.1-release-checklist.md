# v0.1 Release Checklist

StreetSpeak AI v0.1 should ship as a local, mock-first trading desk with a fixture-only Robinhood readiness panel. Do not claim live broker connectivity, investment advice, or trade recommendation behavior.

## Local Setup

- Run `pnpm install`.
- Run `pnpm format`.
- Run `pnpm build`.
- Run `pnpm test`.
- Run `pnpm lint`.
- Run `pnpm typecheck`.
- Run `git diff --check`.

## Demo Commands

- `show my portfolio`
- `what is HOOD trading at`
- `buy 5 HOOD`
- `sell 2 SOFI`
- `build a limit order to buy 3 AAPL at 175`
- `buy $500 of HOOD`

## Safety Checks

- First-run onboarding must be required before the mock demo opens.
- Mock mode must be locked on.
- Live trading must be unavailable.
- Generic confirmations such as `yes`, `do it`, or `confirmed` must be rejected.
- The exact confirmation phrase and code must be required before mock submission.
- Unsupported notional commands must not create final tickets or convert dollars into shares.
- Receipts and audit exports must remain local-only.
- Robinhood adapter status must show `transport: none`.
- Robinhood adapter status must show `requiresCredentials: false`.
- Robinhood order review, placement, cancel order, and live execution must remain unavailable.

## Screenshot Candidates

- First-run onboarding acknowledgement.
- Mock trading desk with portfolio command output.
- Exact confirmation challenge after `buy 5 HOOD`.
- Generic confirmation rejection.
- Mock broker response after exact-code confirmation.
- Local receipt export controls.
- `v0.1 Mock Demo` status panel.
- `Robinhood Read-Only Fixture Explorer` showing fixture-only status and no live connection.

## Public Claim Boundaries

- Do not claim live trading.
- Do not claim a real Robinhood connection.
- Do not claim real account data or real market data.
- Do not claim broker login support.
- Do not claim order review, order placement, or cancel-order support for Robinhood.
- Do not claim investment advice, trade recommendations, signals, or automated trading.
- Do describe the project as local-first, mock-first, self-directed, auditable, and safety-gated.
